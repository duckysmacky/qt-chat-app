# QT Chat App

Простое чат-приложение, написанное на C++ с использованием фреймворка QT.

## О проекте

Это простой проект клиент-серверного чат-приложения, которое поддерживает аккаунты (регистрацию/вход), приватные чаты и группы в формате текстовых сообщений. Оно работает поверх чистого TCP-соединения и использует собственные пакеты для внутреннего обмена данными.

Это учебный проект, который в основном разрабатывается как университетское задание, но при этом остаётся pet-проектом, в который вкладываются усилия, чтобы сделать функциональный продукт и получить качественный практический опыт.

Документацию по исходному коду можно посмотреть на [github pages проекта](https://duckysmacky.github.io/qt-chat-app).

## Сборка

Проект использует CMake в качестве системы сборки. Также для удобства предусмотрен файл `CMakePresets.json`.

### Windows

На Windows проект использует MSVC как компилятор и NMake как генератор.
Пресеты для Windows:

- `windows-msvc-debug`
- `windows-msvc-release`

Сборка с одним из этих профилей **обязательна**, так как они задают необходимые параметры проекта, например `$QT_PREFIX_PATH`.

#### 1. Активируйте окружение Visual Studio

Чтобы собрать проект на Windows, перед сборкой нужно активировать окружение Visual Studio, запустив `scripts/devshell.bat`.
Этот скрипт настраивает переменные окружения, необходимые для использования компилятора MSVC на Windows.

```bash
.\scripts\devshell.bat
```

#### 2. Сконфигурируйте проект с помощью CMake

```bash
cmake --preset=<preset>
```

#### 3. Соберите проект с помощью CMake

Собрать и клиент, и сервер:

```bash
cmake --build --preset=<preset>
```

Или собрать конкретную цель:

```bash
cmake --build --preset=<preset> --target <target>
```

Где:
- `target`: `client` или `server`

### Linux/macOS

На Linux/macOS проект использует GCC как компилятор и Makefile как генератор.

Пресеты Linux/macOS:

- `linux-gcc-debug`
- `linux-gcc-release`

Сборка с одним из этих пресетов **необязательна**, так как на этих платформах директория QT по умолчанию уже задана. Но они всё равно оставлены для удобства.

#### 1. Настройка проекта

##### Без пресета:

```bash
cmake -S . -B build
```

##### С пресетом:

```bash
cmake --preset=<preset>
```

#### 2. Сборка проекта

##### Без пресета:

```bash
cmake --build build/
```

##### С пресетом:

```bash
cmake --build --preset=<preset>
```

##### Конкретная цель:

```bash
cmake --build --preset=<preset> --target <target>
cmake --build build/ --target <target>
```

Где:
- `target`: `client` или `server`

### Кроссплатформенная сборка через Just

Для упрощения сборки в проекте есть `Justfile`. Если хотите его использовать, убедитесь, что `just` установлен в системе.

- На Windows: `just` автоматически запускает сборку через `scripts/devshell.bat` (окружение MSVC + Qt)
- На Linux/macOS: `just` собирает проект обычным образом

#### Сборка проекта

```bash
just build <preset> <target>
```

Где:
- `preset`: `debug` или `release`
- `target`: `client`, `server` или `all`

Либо можно оставить параметры пустыми, чтобы собрать всё в профиле `debug`.

#### Очистка

```bash
just clean
```

## Запуск

Собранные бинарные файлы будут находиться в директории `build/`, в подпапках `build/release` или `build/debug` соответственно.

### Клиент

Просто запустите `client_app` из директории `build/release` или `build/debug`.

### Серверный бинарник

Для сервера требуется запущенная база данных PostgreSQL.

#### 1. Запустите PostgreSQL

Её можно запустить одной из следующих команд:

```bash
just db
docker compose up -d db
```

#### 2. Запустите сервер

Просто запустите `server_app` из директории `build/release` или `build/debug`.

### Образ сервера

Сервер можно запустить внутри Docker-контейнера. Файл `docker-compose.yml` автоматически соберёт образ и запустит контейнер.

```bash
docker compose up -d
```

При необходимости можно собрать образ вручную:

```bash
docker build -t chat-app-server:latest .
```
